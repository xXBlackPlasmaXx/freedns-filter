# FreeDNS Filter Helper

Scrape FreeDNS public domains and classify them with Lightspeed category lookups.

## What this project does

1. Collects domains from FreeDNS (`scripts/freedns-scrape.js`) or from your own file/list.
2. Looks up each host through the Lightspeed websocket endpoint (`src/index.js`).
3. Splits results into allowed vs blocked based on `config/categories.json`.
4. Tracks unseen category IDs in `config/new-categories.json`.
5. Merges new runs into existing output JSON by host and auto-commits changes when possible.

## Quick start

```bash
npm install
npm run cli
```

Helpful commands:

```bash
npm run cli:help
npm run scrape:freedns
npm run clean-output
```

## CLI usage (`npm run cli`)

Interactive by default:

```bash
npm run cli
```

Non-interactive example:

```bash
npm run cli -- --start 1 --end 2 --cookie "DNSID=...; ..." --no-prompt
```

File-driven check (skip scraping):

```bash
npm run cli -- --domains-file ./data/domains.txt --no-prompt
```

### Flags

| Flag | Description |
|------|-------------|
| `-s, --start <page>` | Start page for FreeDNS scrape (default: `1`) |
| `-e, --end <page>` | End page for FreeDNS scrape (default: `start`) |
| `-c, --cookie <str>` | FreeDNS session cookie |
| `-d, --domains-file <path>` | Read domains from file instead of scraping |
| `--no-prompt` | Run with defaults / provided flags only |
| `-h, --help` | Show CLI help |

## Standalone scripts

### `npm run scrape:freedns`

Only scrapes FreeDNS and writes a domain list.

Defaults:

- Registry: `https://freedns.afraid.org/domain/registry/`
- Sort: `2` (Status, Age)
- Delay: `1500` ms
- Output: `data/freedns-public.txt`

### `npm start`

Runs bulk lookup directly (`src/index.js`), without the interactive CLI.

Input order:

1. `LS_DOMAINS` env var (comma/newline-separated)
2. `LS_DOMAINS_FILE` env var (file path)
3. Positional args: `node src/index.js "a.com,b.com" ./my-domains.txt`
4. `data/domains.txt`
5. Legacy fallback: `./domains.txt` (repo root)

Examples:

```bash
npm start
LS_DOMAINS="example.com,wikipedia.org" npm start
LS_DOMAINS_FILE=./data/freedns-public.txt npm start
```

### `npm run clean-output`

Reads `output/results.json` and writes sorted hosts to `output/allowed-domains.txt`.

## Configuration

### Categories (`config/categories.json`)

Each item uses:

- `CategoryNumber` (string or number)
- `CategoryName` (display label)
- `Allow` (`true`/`1` means allowed; everything else is blocked)

### New categories (`config/new-categories.json`)

Unknown category IDs discovered during lookups are auto-added here with sample hosts:

```json
{
  "21": {
    "CategoryNumber": "21",
    "CategoryName": "Unknown",
    "Allow": false,
    "examples": ["some-domain.tld"]
  }
}
```

Promote reviewed entries into `config/categories.json` when you want them enforced.

## Output files

| File | Description |
|------|-------------|
| `data/freedns-public.txt` | Scraped domains (one per line) |
| `output/results.json` | Allowed lookup results (`allowed === true`) |
| `output/blocked.json` | Blocked and errored lookup results |
| `output/allowed-domains.txt` | Allowed host list generated by `clean-output` |
| `config/new-categories.json` | Newly observed categories |

`results.json` and `blocked.json` are merged with existing data by `host` on each run.

## Environment variables

### Lookup / paths

| Variable | Default | Purpose |
|----------|---------|---------|
| `LS_CONCURRENCY` | `5` | Parallel domain lookups |
| `CONCURRENCY` | *(none)* | Fallback for `LS_CONCURRENCY` |
| `LS_TIMEOUT_MS` | `5000` | Per-host lookup timeout |
| `LS_DOMAINS` | *(empty)* | Inline domains for `npm start` |
| `LS_DOMAINS_FILE` | `data/domains.txt` | Domains file for `npm start` |
| `LS_CONFIG_DIR` | `config/` | Config directory override |
| `LS_DATA_DIR` | `data/` | Data directory override |
| `LS_OUTPUT_DIR` | `output/` | Output directory override |

### FreeDNS scrape

| Variable | Default | Purpose |
|----------|---------|---------|
| `FREEDNS_REGISTRY_URL` | Default registry URL | Registry endpoint |
| `FREEDNS_SORT` | `2` | Sort parameter |
| `FREEDNS_QUERY` | *(empty)* | Search query |
| `FREEDNS_START_PAGE` | `1` | Start page |
| `FREEDNS_END_PAGE` | `FREEDNS_MAX_PAGES` or `1` | End page |
| `FREEDNS_MAX_PAGES` | `1` | Fallback end-page value |
| `FREEDNS_DELAY_MS` | `1500` | Delay between pages (ms) |
| `FREEDNS_OUTPUT_FILE` | `data/freedns-public.txt` | Scrape output path |
| `FREEDNS_COOKIE` | *(empty)* | Session cookie |
| `FREEDNS_UA` | Built-in UA | User-Agent header |
| `FREEDNS_ACCEPT_LANGUAGE` | `en-US,en;q=0.9` | Accept-Language header |

## Auto-commit behavior

After successful bulk checks (`npm run cli` or `npm start`), the app attempts to:

1. Bump `package.json` patch version.
2. Stage `output/`, `config/`, `data/`, and `package.json`.
3. Commit with message: `results: +<allowed> allowed, +<blocked> blocked (v<version>)`.

If git is unavailable or there are no commit-ready changes, it logs `Auto-commit skipped` and continues.

## Project structure

```text
src/
  index.js              Lookup engine and bulk runner
  utils.js              Shared helpers (IO, merge, git auto-commit)
scripts/
  cli.js                Interactive/non-interactive scrape + classify flow
  freedns-scrape.js     Standalone FreeDNS scraper
  clean-output.js       Exports allowed hosts to text file
config/
  categories.json       Main category rules
  new-categories.json   Auto-discovered category IDs
data/
  domains.txt           Default input domains
  freedns-public.txt    Scraped domains
output/
  results.json          Allowed hosts
  blocked.json          Blocked/errored hosts
  allowed-domains.txt   Allowed hosts as plain text
```
